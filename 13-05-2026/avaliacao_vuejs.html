1. Objetivo

Construir um pequeno CRUD (Create / Read / Update / Delete) de Cadastro de Alunos em Vue 3 com <script setup> (Composition API), aplicando obrigatoriamente todas as diretivas vistas em sala: ref(), interpolação {{ }}, v-if, v-else, v-show, v-for (mais v-model e @click já usados).

Cada aluno cadastrado deve conter, no mínimo, os seguintes campos:

4. Requisitos funcionais

RF — O que o sistema deve fazer
  • RF1 (Create): permitir cadastrar um novo aluno preenchendo nome, matrícula, curso e status (ativo).
  • RF2 (Read): listar todos os alunos cadastrados, mostrando posição (#1, #2, ...), nome, matrícula, curso e status.
  • RF3 (Update): permitir editar qualquer aluno já cadastrado e salvar as alterações.
  • RF4 (Delete): permitir excluir um aluno da lista.
  • RF5: exibir o total de alunos cadastrados no topo da tela.
  • RF6: quando a lista estiver vazia, mostrar uma mensagem amigável no lugar da tabela/lista.
  • RF7: o formulário de cadastro/edição deve poder ser mostrado/ocultado por um botão.

5. Requisitos técnicos — uso OBRIGATÓRIO das diretivas

ref() — para a lista de alunos e para os campos do formulário (nome, matrícula, curso, ativo).
Interpolação {{ }} — para exibir o total de alunos e os dados de cada aluno na lista.
v-if / v-else — para alternar entre exibir a lista de alunos e a mensagem de lista vazia; e/ou para alternar o título/botão entre os modos "Cadastrar" e "Editar".
v-show — para mostrar/ocultar o formulário ao clicar em "Mostrar/Ocultar formulário".
v-for — para percorrer a lista de alunos. Use :key com o id do aluno.
v-model + @click — para os campos do formulário e para os botões de Salvar, Cancelar, Editar e Excluir.
Dica de implementação: use uma variável const editandoId = ref(null) para saber se está cadastrando um aluno novo (null) ou editando um já existente (id do aluno). Isso resolve o v-if/v-else do título "Cadastrar" vs "Editar" e a lógica do botão Salvar.
