1. Objetivo

Construir uma mini-aplicação Vue 3 + Vuetify usando Vue Router e componentes reutilizáveis, contendo um CRUD completo (Create / Read / Update / Delete) que opere sobre uma lista mantida em variáveis locais (sem backend, sem banco de dados — os dados moram em memória, como uma ref ou reactive). A aplicação deve girar em torno de um tema específico, escolhido por você no início da atividade.

Escolha um tema para sua aplicação e se comprometa com ele (preencha o campo "Tema escolhido" no cabeçalho). Sugestões abaixo, mas você pode propor o seu:

(Outro) crie o seu — desde que tenha pelo menos 5 campos com tipos variados (string, number, boolean e/ou enum)
Regra do tema: o tema escolhido precisa aparecer de forma visível na aplicação — nas cores, no nome no v-app-bar, nos ícones do menu, no texto da página "Sobre" e nos rótulos dos campos do formulário. Não vale escolher "Lanchonete" e exibir "Cadastro de itens" na tela.

3. Requisitos funcionais

RF — O que o sistema deve fazer
  • RF1 — Navegação: a aplicação deve ter no mínimo 3 rotas distintas, navegáveis por um menu (drawer ou app-bar) usando router-link ou :to.
  • RF2 — Create: permitir cadastrar um novo item do tema escolhido por meio de um formulário Vuetify (v-text-field, v-select, v-switch/v-checkbox, etc.).
  • RF3 — Read: listar todos os itens cadastrados em uma página própria, usando componentes Vuetify (ex.: v-list, v-data-table ou v-card em grid).
  • RF4 — Update: permitir editar qualquer item já cadastrado e salvar as alterações.
  • RF5 — Delete: permitir excluir um item (com confirmação, via v-dialog ou confirm()).
  • RF6 — Estado global local: os dados precisam sobreviver à troca de rota — ao sair da página de lista para o formulário e voltar, os itens cadastrados continuam lá.
  • RF7 — Página "Sobre": rota com nome da dupla, matrículas e descrição curta do tema.
  • RF8 — Lista vazia: mostrar uma mensagem amigável quando ainda não houver itens cadastrados (e um botão para cadastrar o primeiro).

/lista Lista de itens (Read) Renderiza todos os itens em v-data-table, v-list ou cards. Cada linha tem botões Editar e Excluir.

Pode ter mais rotas? Pode. /lista + /cadastro + /sobre + / = 4 rotas, já passa do mínimo. Se quiser criar uma rota extra (estatísticas, "favoritos", filtro por categoria, etc.), pontuação só sobe na qualidade.

6. Requisitos técnicos — checklist obrigatório

Projeto criado com bun create vuetify (ou equivalente) — Vue 3 + Vite + Vuetify.
Vue Router configurado com no mínimo 3 rotas distintas, registradas em src/router/index.js (ou .ts).
Navegação por menu (drawer ou app-bar) usando <router-link> ou :to="..." em v-btn/v-list-item.
<router-view /> no App.vue, renderizando a página correta para cada rota.
Pelo menos 2 componentes reutilizáveis próprios em src/components/ (ex.: ItemCard.vue, ItemForm.vue, EstatisticasBox.vue) — recebendo props e/ou emitindo events.
Tema Vuetify customizado em src/plugins/vuetify.js: pelo menos primary e secondary alteradas para combinar com o tema escolhido.
CRUD completo: Create, Read, Update e Delete, todos funcionando sobre uma lista ref([]) ou reactive([]).
Estado preservado entre rotas: a lista mora em um arquivo compartilhado (ex.: src/store/itens.js exportando um ref) ou em um composable (useItens.js) — não dentro do setup de uma página específica.
Uso natural de pelo menos 5 componentes Vuetify (ex.: v-app-bar, v-navigation-drawer, v-card, v-text-field, v-btn, v-data-table, v-dialog, v-select, v-switch...).
Sem erros vermelhos no console do navegador ao usar a aplicação.

7. Como compartilhar o estado entre rotas (dica essencial)

Como cada página é um componente diferente, declarar const itens = ref([]) dentro de Lista.vue e dentro de Cadastro.vue cria duas listas separadas. A forma mais simples de compartilhar é criar um arquivo só pra isso: 

// src/store/itens.js
import { ref } from 'vue';

export const itens = ref([]);  // uma única lista, importada por todas as páginas

export function salvar(item) {
  if (item.id) {
    const i = itens.value.findIndex(x => x.id === item.id);
    itens.value[i] = { ...item };
  } else {
    itens.value.push({ ...item, id: Date.now() });
  }
}

export function excluir(id) {
  itens.value = itens.value.filter(x => x.id !== id);
}

Depois, em qualquer página: import { itens, salvar, excluir } from '@/store/itens'. Pronto, mesma lista em toda a aplicação.

Alternativa avançada: se quiser, dá para usar Pinia. Mas não é obrigatório — o arquivo compartilhado acima já cumpre o RF6.
